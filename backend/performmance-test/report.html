4. Findings

  • The system maintained stable performance under normal and moderate concurrent workloads with low error rates and acceptable response times.
  • Search, Wishlist, Trending Deal, Price Alert, and Price History APIs demonstrated good scalability and remained responsive under increasing load.
  • Compare Price API showed noticeable performance degradation under higher concurrency levels and became the primary performance bottleneck.
  • Database operations and comparison processing logic may contribute to increased latency during peak traffic conditions.
  • The current environment (4 vCPUs / 16GB RAM) was able to support moderate concurrent traffic while maintaining overall service stability.

5. Recommendations

  • Add Rate Limiting to protect backend services from excessive concurrent requests.
  • Enable Caching for Search, Trending Deal, and Price History endpoints.
  • Optimize database queries by improving indexes and reducing unnecessary joins.
  • Apply Horizontal Scaling to support larger traffic volumes.
  • Tune Connection Pool configuration to improve throughput under load.
  • Refactor Compare Price processing logic and reduce synchronous operations.
  • Continue monitoring and perform periodic stress testing after optimization.
